# 今天吃什么

一个不依赖框架和后端的两层静态随机用餐选择器，可直接部署到 GitHub Pages。

## 抽取规则

1. 先从一级分类中随机抽取。
2. 一级分类在抽中后的滚动 24 小时内不会再次进入候选池。
3. 再从选中分类的二级选项中随机抽取具体结果。
4. 如果所有含有二级选项的一级分类都处于 24 小时锁定期，网页会停止抽取并显示最早解锁时间，不会自动放宽规则。

## 功能

- 两层随机：一级分类 → 二级选项
- 一级分类 24 小时内严格不重复
- 内置 14 个一级分类和 85 个二级选项
- 在网页中添加、删除一级分类和二级选项
- 通过 TXT 文件导入个性化列表
- 导入采用整表替换，不会与旧列表合并
- 导入成功后清空旧列表、抽取历史和 24 小时锁定
- 保存最近抽取记录
- 撤销上一条记录会同时解除对应的分类锁定
- 清空记录会解除全部分类锁定
- 浅色 / 深色模式
- 数据保存在浏览器 `localStorage`
- 手机和桌面端自适应

## TXT 导入格式

使用 UTF-8 编码的 `.txt` 文件。一级分类使用方括号，后续每一行是一个二级选项：

```text
[烤鱼]
重麻辣鮰鱼
青花椒鮰鱼
藤椒凌波鱼

[日料与刺身]
三文鱼刺身
综合刺身拼盘
```

规则：

- 支持半角方括号 `[分类]` 和全角方括号 `【分类】`
- 空行会被忽略
- 以 `#` 或 `//` 开头的行会被视为注释
- 一级分类不能重复，且必须至少包含一个二级选项
- 同一分类内的重复二级选项会自动去重
- 文件验证通过后才会覆盖浏览器中的旧数据
- 文件上限为 1 MB

仓库内的 `meal-list-example.txt` 可直接下载后修改。

## 本地打开

直接双击 `index.html` 即可。也可以启动一个简单的静态服务器：

```bash
python -m http.server 8000
```

然后打开 `http://localhost:8000`。

## 部署到 GitHub Pages

1. 将文件放在仓库根目录。
2. 打开仓库的 **Settings → Pages**。
3. 在 **Build and deployment** 中选择 **Deploy from a branch**。
4. Branch 选择 `main`，目录选择 `/ (root)`，保存。

## 修改默认清单

编辑 `app.js` 顶部的 `DEFAULT_CATEGORIES`。已经自定义过列表的浏览器会优先使用本地数据；点击网页中的“恢复默认列表”可以载入代码内的默认清单。
