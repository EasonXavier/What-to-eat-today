# 今天吃什么

## Version

Current version: `1.2.1`. See [VERSIONING.md](./VERSIONING.md) for the `a.b.c` increment policy.

一个不依赖框架和后端的两层随机用餐选择器，可直接部署到 GitHub Pages。

## 抽取规则

1. 先从大类中随机抽取。
2. 大类在抽中后的滚动 24 小时内不会再次进入候选池。
3. 再从该大类中随机抽取一个具体菜式。
4. 所有大类都处于锁定期时，网页会停止抽取并显示最早可用时间，不会自动放宽规则。

## 功能

- 两层随机：大类 → 具体菜式
- 大类 24 小时内严格不重复
- 内置 16 个大类和 96 个外卖友好选项
- 默认选项采用“酸辣鱼”“麻辣烤鱼”等适中粒度
- 添加或删除大类和具体选项
- TXT 整表导入，不与原菜单合并
- TXT 导出当前菜单
- 导出的 TXT 自带完整格式说明、粒度建议和 AI 客制化提示
- 导入成功后清空旧菜单、抽取记录和 24 小时锁定
- 撤回上一次结果或清空全部记录
- 浅色与深色模式
- 数据保存在浏览器 `localStorage`
- 使用 Apple 平台系统字体栈
- 分别适配 iPhone、iPad 和桌面浏览器

## TXT 导入与导出

网页中的“导出列表”会生成 UTF-8 TXT 文件。文件包括：

- 当前全部大类和具体选项
- 可重新导入的格式说明
- 食物名称粒度建议
- 可直接交给其他 AI 使用的客制化提示

修改后通过“导入列表”选择该文件。文件完整验证通过后才会覆盖原菜单；格式错误不会影响现有数据。

## 本地打开

直接打开 `index.html`，或者启动静态服务器：

```bash
python -m http.server 8000
```

然后访问 `http://localhost:8000`。

## 部署到 GitHub Pages

1. 将文件放在仓库根目录。
2. 打开仓库的 **Settings → Pages**。
3. 在 **Build and deployment** 中选择 **Deploy from a branch**。
4. Branch 选择 `main`，目录选择 `/ (root)`，保存。

## 默认菜单

默认菜单位于 `default-list.js`。该文件负责：

- 首次访问时写入当前默认菜单
- 自动升级未修改过的旧版默认菜单
- 当浏览器保存的是空数组、损坏数据或没有任何可抽取菜式时，重新写入完整默认菜单
- 保留仍包含有效菜式的用户自定义菜单
- 向主程序提供“恢复默认”所使用的数据

已经客制化且仍包含有效菜式的菜单不会被自动覆盖。
