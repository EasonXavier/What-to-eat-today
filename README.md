# 今天吃什么

一个不依赖框架和后端的两层静态随机用餐选择器，可直接部署到 GitHub Pages。

## 抽取规则

1. 先从一级分类中随机抽取。
2. 一级分类在抽中后的滚动 24 小时内不会再次进入候选池。
3. 再从选中分类的二级选项中随机抽取具体结果。
4. 如果所有含有二级选项的一级分类都处于 24 小时锁定期，网页会停止抽取并显示最早解锁时间，不会自动放宽规则。

## 功能

- 两层随机：一级分类 → 二级选项
- 一级分类 24 小时内严格不重复
- 在网页中添加、删除一级分类和二级选项
- 保存最近抽取记录
- 撤销上一条记录会同时解除对应的分类锁定
- 清空记录会解除全部分类锁定
- 浅色 / 深色模式
- 数据保存在浏览器 `localStorage`
- 手机和桌面端自适应
- 自动迁移旧版单层备选清单

## 本地打开

直接双击 `index.html` 即可。也可以启动一个简单的静态服务器：

```bash
python -m http.server 8000
```

然后打开 `http://localhost:8000`。

## 部署到 GitHub Pages

1. 将文件放在仓库根目录。
2. 打开仓库的 **Settings → Pages**。
3. 在 **Build and deployment** 中选择 **Deploy from a branch**。
4. Branch 选择 `main`，目录选择 `/ (root)`，保存。

## 修改默认清单

编辑 `app.js` 顶部的 `DEFAULT_CATEGORIES`：

```js
const DEFAULT_CATEGORIES = [
  { id: 'noodles', name: '面食类', items: ['牛肉面', '馄饨'] }
];
```

已经在浏览器中使用过网页时，浏览器保存的清单优先于代码默认值。点击网页中的“恢复示例”可以重新载入代码内的默认清单。
